# LinkedIn Parser

Парсер профилей LinkedIn для учебных целей. Работает без авторизации, использует бесплатные прокси-серверы через ProxyScrape API и соблюдает лимиты LinkedIn.

## Особенности

- ✅ Парсинг без авторизации (только публичные профили)
- ✅ Автоматическая ротация прокси-серверов
- ✅ Соблюдение лимитов LinkedIn (задержки между запросами)
- ✅ Использование только бесплатных библиотек
- ✅ Сохранение данных в структурированном JSON формате

## Требования

- Python 3.8 или выше
- Интернет-соединение

## Установка

1. Клонируйте репозиторий или скачайте файлы проекта

2. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Использование

### Парсинг одного профиля

```bash
python main.py https://www.linkedin.com/in/username
```

### Парсинг из файла

Создайте файл `urls.txt` со списком URL (по одному на строку):

```
https://www.linkedin.com/in/user1
https://www.linkedin.com/in/user2
https://www.linkedin.com/in/user3
```

Запустите парсер:

```bash
python main.py -f urls.txt
```

### Дополнительные опции

```bash
# Указать другой файл конфигурации
python main.py -c my_config.json https://www.linkedin.com/in/username

# Указать директорию для сохранения результатов
python main.py -o results https://www.linkedin.com/in/username
```

## Конфигурация

Настройки находятся в файле `config.json`. Основные параметры:

- **proxy.enabled** - включить/выключить использование прокси
- **proxy.max_proxies** - максимальное количество прокси для использования
- **rate_limiting.min_delay** - минимальная задержка между запросами (секунды)
- **rate_limiting.max_delay** - максимальная задержка между запросами (секунды)
- **linkedin.timeout** - таймаут запросов (секунды)
- **linkedin.max_retries** - количество повторных попыток при ошибке

## Формат выходных данных

Результаты сохраняются в директории `output/` в формате JSON. Каждый профиль сохраняется в отдельный файл `{publicIdentifier}.json`.

Структура данных соответствует предоставленному примеру и включает:
- Базовую информацию (имя, заголовок, фото, локация)
- Опыт работы (experience)
- Образование (education)
- Навыки (skills)
- Сертификаты, проекты, волонтерство
- И другие данные профиля

## Ограничения

⚠️ **Важно понимать:**

1. **LinkedIn активно блокирует парсинг** - без авторизации доступна только ограниченная информация из публичных профилей
2. **Бесплатные прокси нестабильны** - многие прокси могут не работать или быть медленными
3. **Соблюдение ToS** - использование парсера может нарушать условия использования LinkedIn
4. **Ограниченные данные** - без авторизации многие данные профиля недоступны

## Логирование

Все операции логируются в файл `linkedin_parser.log` и выводятся в консоль.

## Устранение неполадок

### Прокси не работают

Если прокси не работают, парсер автоматически переключится на прямые запросы. Вы можете отключить прокси в `config.json`, установив `proxy.enabled: false`.

### Ошибка 403 (Forbidden)

LinkedIn может блокировать запросы. Попробуйте:
- Увеличить задержки между запросами в `config.json`
- Использовать больше прокси
- Проверить, что профиль публичный

### Мало данных в результате

Без авторизации LinkedIn показывает только публичную информацию. Многие данные (полный опыт работы, навыки, рекомендации) могут быть недоступны.
